This is triggered when the location of a variable changes between program points (blocks and values). Specifically, the updateVar method is called when the location of a variable changes between blocks or between values within a block. It checks if the variable's current location can be merged with its pending entry (from the previous location). So writing out pending entries allows the location list for each variable to accurately reflect all the changes in its location through the course of the program's execution. The change in location triggers writing out the entry so that different live ranges are properly delimited in the final location lists.
